# ü§ù Contributing to Git & GitHub Learning Guide

Thank you for your interest in contributing! This guide is meant to help people learn Git and GitHub, and your contributions make it better for everyone.

## üìã How to Contribute

### Reporting Issues

Found a typo, error, or have a suggestion? Please open an issue!

1. Go to the Issues tab
2. Click "New Issue"
3. Choose the appropriate template
4. Provide clear description

### Suggesting Improvements

Have ideas to make this guide better?
- Open an issue with the "enhancement" label
- Describe your idea clearly
- Explain why it would be helpful

### Submitting Changes

1. **Fork the repository**
   ```bash
   # Click Fork button on GitHub
   ```

2. **Clone your fork**
   ```bash
   git clone https://github.com/your-username/learn-git-github.git
   cd learn-git-github
   ```

3. **Create a branch**
   ```bash
   git checkout -b fix/typo-in-readme
   # or
   git checkout -b feature/add-new-section
   ```

4. **Make your changes**
   - Edit files
   - Add new content
   - Fix errors
   - Improve explanations

5. **Commit your changes**
   ```bash
   git add .
   git commit -m "fix: Correct typo in README"
   ```

6. **Push to your fork**
   ```bash
   git push -u origin fix/typo-in-readme
   ```

7. **Create Pull Request**
   - Go to your fork on GitHub
   - Click "Compare & pull request"
   - Fill out the PR description
   - Submit!

## ‚úçÔ∏è Writing Guidelines

### Commit Messages

Follow conventional commits:
- `fix:` for bug fixes
- `feat:` for new features
- `docs:` for documentation
- `style:` for formatting
- `refactor:` for code restructuring

### Content Style

- **Clear and simple**: Write for beginners
- **Examples**: Include practical examples
- **Commands**: Show actual commands with explanations
- **Visual**: Reference diagrams and screenshots
- **Friendly**: Keep tone welcoming and encouraging

### Markdown Formatting

- Use proper heading hierarchy
- Include code blocks with syntax highlighting
- Add emojis sparingly (they're already in the guide)
- Keep line length reasonable (80-100 chars)

## üñºÔ∏è Adding Images

If you're adding images:

1. Place them in appropriate `images/` subdirectory
2. Follow naming conventions (lowercase, hyphens)
3. Optimize for web (keep file size reasonable)
4. Update the relevant README to reference the image
5. Add alt text in Markdown
