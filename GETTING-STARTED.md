## ‚ö° Quick Start (5 minutes)

If you want to get started immediately:

```bash
# 1. Install Git (if not installed)
# Visit: https://git-scm.com/downloads

# 2. Configure Git
git config --global user.name "Your Name"
git config --global user.email "your.email@example.com"

# 3. Create your first repository
mkdir my-first-repo
cd my-first-repo
git init

# 4. Create a file
echo "# My First Repo" > README.md

# 5. Commit it
git add README.md
git commit -m "Initial commit"

# 6. View history
git log
```

You just used Git!

## üéì Learning Tips

### 1. Practice Daily
- Use Git for all your projects
- Try new commands regularly
- Make mistakes and learn from them

### 2. Start Small
- Don't try to learn everything at once
- Master basics before advanced topics
- Build confidence gradually

### 3. Use Real Projects
- Apply Git to actual work
- Practice on personal projects
- Learn by doing

### 4. Read Error Messages
- Git error messages are helpful
- They often tell you what to do
- Don't panic - most issues are fixable

### 5. Use Help
```bash
# Get help for any command
git help <command>
git <command> --help

# Example
git help commit
git commit --help
```

## üÜò Getting Help

### Common Issues

**"Git is not recognized"**
- Git is not installed or not in PATH
- Solution: Install Git and restart terminal

**"Permission denied"**
- Authentication issue with GitHub
- Solution: Set up SSH keys or use personal access token

**"Merge conflict"**
- Two branches changed the same code
- Solution: See [Branching & Merging](./04-branching-merging/README.md)

### Where to Get Help

1. **This Guide**: Check relevant sections
2. **Git Help**: `git help <command>`
3. **GitHub Docs**: https://docs.github.com
4. **Stack Overflow**: Search for specific errors
5. **Git Community**: https://git-scm.com/community
