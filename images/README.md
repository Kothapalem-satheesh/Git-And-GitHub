## ğŸ¨ Creating Images

### Diagrams

You can create diagrams using:
- **Draw.io** (diagrams.net) - Free, web-based
- **Lucidchart** - Professional diagrams
- **Excalidraw** - Hand-drawn style
- **Mermaid** - Code-based diagrams (can be embedded in Markdown)

### Screenshots

Take screenshots of:
- Terminal/command line output
- GitHub interface
- Git commands in action
- Code editors showing Git integration

## ğŸ“ Image Guidelines

### Naming Convention
- Use lowercase with hyphens: `git-status-output.png`
- Be descriptive: `merge-conflict-example.png`
- Include context: `github-create-repo-button.png`

### Image Specifications
- **Format**: PNG (for diagrams) or JPG (for photos)
- **Size**: Optimize for web (under 500KB when possible)
- **Resolution**: 1920x1080 or higher for screenshots
- **Aspect Ratio**: Maintain consistent ratios

### Accessibility
- Add alt text in Markdown: `![Description](path/to/image.png)`
- Use high contrast for diagrams
- Include text labels on diagrams

## ğŸ”— Using Images in Documentation

In your Markdown files, reference images like this:

```markdown
![Git Timeline Diagram](../images/diagrams/git-timeline.png)
```

Or with a caption:

```markdown
<figure>
  <img src="../images/diagrams/git-timeline.png" alt="Git commit timeline">
  <figcaption>Timeline showing Git commits over time</figcaption>
</figure>
```
